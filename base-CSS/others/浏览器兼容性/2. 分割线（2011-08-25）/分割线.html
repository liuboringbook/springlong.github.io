浏览器兼容——分割线

问题二：

  1. 在浏览器中，<hr />标签（“水平线”）的默认宽度为100%，高度为2px，表现为上下共2px的凹槽边框，边框的颜色为灰色!
  2. 水平线的水平对齐方式在IE和Opera浏览器中默认为居中对齐；而在Firefox、Chrome和Safari浏览器中，则默认为左对齐。
  3. 在IE和Opera浏览器中，通过设置text-align属性为left可以实现水平线居左对齐的效果，而其他浏览器中默认的就是左对齐；在IE6/7和Opera浏览器中，通过设置text-align属性为right可以实现水平线居右对齐的效果，而在其他浏览器中则无法实现，另外设置text-align为right或center都将导致在IE8/9中的水平线呈居左对齐显示；所有浏览器中都允许使用margin-left来设置水平线的左侧边距，从而可以通过调整水平线的左侧边距来达到水平线居右对齐的效果（仅限父级容器固定宽度的情况下 ）！
  4. 在IE8/9、Chrome、Safari以及Opera浏览器中，可通过background-color属性设置水平线的表现颜色，以及通过border属性设置外围的边框样式（包括样式、粗细和颜色）；而在IE6/7浏览器中，可以通过background-color属性设置水平线的表现颜色，以及通过border属性设置外围的边框样式，但是依旧存在四周各1像素的凹槽边缘，可以通过color属性设置该边缘的颜色。(说明：通过color属性在IE6/7中设置的是整个水平线的颜色！）
  5. 当设置<hr />标签的height属性大于2时，在Firefox、IE6/7浏览器中，水平线的总高度（包含边框）即为height属性值；而在IE8/9、Chrome、Safari、Opera浏览器中，水平线的总高度为height属性值+边框！
  6. 在IE6/7浏览器中，水平线默认拥有7像素的上下边距，如果设置"margin-top:10px;"的话，则水平线的最终顶部边距将为17px，而在其他浏览器中则不会出现该问题；另外在IE6/7浏览器中，水平线的高度（包括height、border和margin）至少有14px，即使你将水平线的高度设置为0。

解决方法： 0. 如果需要保持水平线的默认样式，则什么也不做，否则，按照后面的3个步骤来进行； 1. 由于各浏览器在实现水平线的高度时有差别，而这差别的造成因素是边框，所以在设置水平线样式之前，应用"border:none;"从而使各浏览器在处理水平线高度时达成一致； 2. 各浏览器在设置水平线颜色时也各有不同，去掉水平线的边框后，我们就可以同时设置水平线的background-color和color这两个属性来实现水平线的颜色了； 3. 对于IE6/7浏览器的上下边距问题，我们在设置其样式时，可以利用相关CSS Hack，将IE6/7中水平线的上下边距在正常的边距值下再减去7px； 4. 如果需要水平线居左对齐，则设置text-align为left即可；如果需要水平线居中对齐，则设置margin-left和margin-right为auto即可；如果需要水平线居右对齐（在父级容器固定宽度的情况下），则针对IE6/7浏览器设置text-align为right，而针对其他浏览器设置margin-left为(父级容器宽度-水平线宽度)的结果值！

下面的水平线在各浏览器中都表现一致，请在不同浏览器下测试结果：

"width:50%; height:15px; border:none; margin-top:0; margin-bottom:0; *margin-top:-7px; *margin-bottom:-7px; margin-left:50%; *margin-left:0; *text-align:right; background:#086A74; color:#086A74;"

说明：使用<hr />标签作为分割线使用时，利用上述的解决方法可以实现各浏览器中的兼容，但是无法实现分割线的自由居右对齐的效果，另外也无法解决IE6/7最小占用高度为14px的问题！
